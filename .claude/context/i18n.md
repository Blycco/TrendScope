# Context: Internationalization (i18n)

> Frontend agent exclusive. No hardcoded user-facing strings anywhere.

## Setup
- Library: svelte-i18n or equivalent
- Translation files: frontend/src/lib/i18n/{locale}.json
- Supported locales: ko (default), en
- Locale detection: user profile settings → browser header fallback

## Rules
- Every user-facing string MUST use a translation key
- No hardcoded Korean or English text in .svelte or .ts files
- Use script to extract and validate missing keys before commit
- Translation key naming: {domain}.{component}.{element}
  e.g. nav.sidebar.trends, modal.quota_exceeded.title, error.auth_required

## Key Categories
```
nav.*          — navigation items
page.*         — page titles and headers
modal.*        — modal titles and messages
error.*        — error messages (paired with error codes)
label.*        — form labels and field names
button.*       — button labels
status.*       — status badges and indicators
admin.*        — admin panel strings
```

## Script Usage
```bash
# Check for missing keys
npm run i18n:check

# Extract hardcoded strings (lint)
npm run i18n:lint
```
